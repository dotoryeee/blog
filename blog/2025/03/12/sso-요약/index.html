Pod Identity vs IRSA в╣ёЖхљьЉю

в╣ёЖхљ ьЋГвфЕ Pod Identity (Pod Identity Agent + EKS Auth API) IAM Role for Service Account (IRSA)
OIDC ВѓгВџЕ ВЌгвХђ X (вХѕьЋёВџћ) O
IAM Role ВЌ░Ж▓░ в░ЕВІЮ Pod Identity AssociationВю╝вАю ServiceAccountВЌљ вДцьЋЉ ServiceAccount Вќ┤вЁИьЁїВЮ┤ВЁўВю╝вАю IAM Role ВЌ░Ж▓░
AWS STS ьўИВХю ВЌгвХђ X (EKS Auth API ьўИВХю) O (AssumeRoleWithWebIdentity)
EKS API Server ВЮИВдЮ Ж│╝ВаЋ Pod Identity AgentЖ░ђ EKS Auth APIвАю ВъљЖ▓Е ВдЮвфЁ ВџћВ▓Г OIDC Provider ьєхьЋ┤ STSВЌљВёю ьєаьЂ░ Ж▓ђВдЮ ьЏё в░юЖИЅ
EKS ВЎИвХђ ВаЉЖи╝ Ж░ђвіЦ ВЌгвХђ X (Agent ВЮўВА┤) O (OIDC ьјўвЇћваѕВЮ┤ВЁў)
AssumeRole ВІюВаљ Pod ВІцьќЅ ВІю AgentЖ░ђ EKS Auth APIвАю ВъљЖ▓Е ВдЮвфЁ ьџЇвЊЮ PodЖ░ђ OIDC ьєаьЂ░Вю╝вАю STS AssumeRoleWithWebIdentity ьўИВХю
ВѓгВџЕ ВўѕВІю EKS вѓ┤вХђ PodЖ░ђ AWS Вёюв╣ёВіц ВаЉЖи╝ (ВёцВаЋ Ж░ёьјИ) OIDC ьјўвЇћваѕВЮ┤ВЁў┬иEKS ВЎИвХђ ВаЉЖи╝ВЮ┤ ьЋёВџћьЋю Ж▓йВџ░

EKS Pod Identity ВЮИВдЮ ьћёвАюВёИВіц

sequenceDiagram
    participant User as ВѓгВџЕВъљ
    participant Kubernetes as Kubernetes Cluster (EKS)
    participant ServiceAccount as Kubernetes ServiceAccount
    participant Pod as Pod (ВЋ▒ В╗еьЁїВЮ┤вёѕ)
    participant Agent as EKS Pod Identity Agent
    participant EKSAuth as EKS Auth API
    participant AWS_Service as AWS Вёюв╣ёВіц (Вўѕ: S3, DynamoDB)

    User->>Kubernetes: Pod ВІцьќЅ ВџћВ▓Г (ServiceAccount ВёцВаЋ ьЈгьЋе)
    Kubernetes->>ServiceAccount: PodВЌљ ServiceAccount ВЌ░Ж▓░ (Pod Identity Association)
    Pod->>Agent: ВъљЖ▓Е ВдЮвфЁ ВџћВ▓Г (AWS_CONTAINER_CREDENTIALS_FULL_URI)
    Agent->>EKSAuth: AssumeRoleForPodIdentity ьўИВХю (OIDC в»ИВѓгВџЕ)
    EKSAuth->>Agent: ВъёВІю AWS ВъљЖ▓Е ВдЮвфЁ в░ўьЎў (AccessKey, SecretKey, Token)
    Agent->>Pod: ВъёВІю ВъљЖ▓Е ВдЮвфЁ ВаёвІг
    Pod->>AWS_Service: IAM Role ЖИ░в░ўВЮў ВЮИВдЮВю╝вАю AWS Вёюв╣ёВіц ВаЉЖи╝
    AWS_Service-->>Pod: ВџћВ▓Гвљю вЇ░ВЮ┤ьё░ в░ўьЎў

IAM Role for Service Account (IRSA) ьћёвАюВёИВіц

sequenceDiagram
    participant Pod as Pod (ВЋаьћївдгВ╝ђВЮ┤ВЁў В╗еьЁїВЮ┤вёѕ)
    participant ServiceAccount as Kubernetes ServiceAccount
    participant AWS_OIDC as AWS OIDC Provider
    participant IAM as AWS IAM Role
    participant STS as AWS Security Token Service (STS)
    participant AWS_Service as AWS Вёюв╣ёВіц (S3, DynamoDB вЊ▒)
    participant Pod_Env as Pod вѓ┤вХђ ьЎўЖ▓й в│ђВѕў

    Pod->>ServiceAccount: OIDC ьєаьЂ░ ВџћВ▓Г
    ServiceAccount->>AWS_OIDC: OIDC ьєаьЂ░ВЮё AWS OIDC ProviderВЌљ ВаёвІг
    AWS_OIDC->>STS: ID ьєаьЂ░ Ж▓ђВдЮ ВџћВ▓Г
    STS->>AWS_OIDC: Ж▓ђВдЮ ВЎёвБї ВЮЉвІх
    ServiceAccount->>IAM: IAM RoleЖ│╝ ВЌ░Ж▓░вљю ServiceAccount ВЮИВдЮ
    IAM->>STS: AssumeRoleWithWebIdentity ьўИВХю (OIDC ьєаьЂ░ ьЈгьЋе)
    STS->>IAM: ВъёВІю AWS в│┤ВЋѕ ВъљЖ▓Е ВдЮвфЁ (AccessKey, SecretKey, Session Token) в░юЖИЅ
    IAM->>Pod_Env: Pod вѓ┤вХђ ьЎўЖ▓й в│ђВѕўВЌљ ВъљЖ▓Е ВдЮвфЁ ВађВъЦ
    Pod->>AWS_Service: AWS Вёюв╣ёВіц ВџћВ▓Г (IAM Role ЖИ░в░ў ВЮИВдЮ)
    AWS_Service-->>Pod: ВџћВ▓Гвљю вЇ░ВЮ┤ьё░ в░ўьЎў

Pod Identity ВІювѓўвдгВўц

ВЃЂьЎЕ в╣ёЖ│а
OIDC Provider ВёцВаЋ ВЌєВЮ┤ ВЮИВдЮьЋўЖ│а ВІХВЮё вЋї OIDC ЖхгВё▒ ВЌєВЮ┤ Pod Identity AssociationвДїВю╝вАю ServiceAccountВЎђ IAM RoleВЮё ВЌ░Ж▓░ьЋа Вѕў ВъѕВЮї
EKS ьЂ┤вЪгВіцьё░ вѓ┤вХђ PodВЌљВёю AWS ВаЉЖи╝ВЮ┤ ьЋёВџћьЋа вЋї вЁИвЊюВЮў Pod Identity AgentЖ░ђ ВъљЖ▓Е ВдЮвфЁВЮё в░юЖИЅьЋўв»ђвАю ьЂ┤вЪгВіцьё░ вѓ┤вХђВЌљВёювДї вЈЎВъЉьЋе (ВЎИвХђ ВаЉЖи╝ вХѕЖ░ђ)
вІеЖИ░ ВъљЖ▓Е ВдЮвфЁВЮ┤ ьЋёВџћьЋа вЋї EKS Auth APIЖ░ђ в░юЖИЅьЋўвіћ ВъёВІю ВъљЖ▓Е ВдЮвфЁВЮђ ВДДВЮђ TTL(Time-To-Live)ВЮё Ж░ђВДђв»ђвАю, в│┤ВЋѕВё▒ВЮ┤ ьќЦВЃЂвље
ьЋўвѓўВЮў IAM RoleВЮё ВЌгвЪг ьЂ┤вЪгВіцьё░ВЌљВёю ВъгВѓгВџЕьЋўЖ│а ВІХВЮё вЋї ВІавб░ Ж┤ђЖ│ёвЦ╝ EKSЖ░ђ Ж┤ђвдгьЋўв»ђвАю IAM Role ВІавб░ ВаЋВ▒Ё ВѕўВаЋ ВЌєВЮ┤ ВЌгвЪг ьЂ┤вЪгВіцьё░ВЌљВёю ВѓгВџЕ Ж░ђвіЦьЋе
STS ьўИВХю ВЌєВЮ┤ ВъљЖ▓Е ВдЮвфЁВЮё в░юЖИЅв░ЏЖ│а ВІХВЮё вЋї Pod Identity AgentЖ░ђ AssumeRoleForPodIdentityвЦ╝ ьўИВХюьЋўВЌг STS ВЌєВЮ┤ ВъљЖ▓Е ВдЮвфЁВЮё Ж░ђВаИВўг Вѕў ВъѕВЮї

Pod Identity в░ЕВІЮВЮђ OIDC ВёцВаЋ ВЌєВЮ┤ Pod Identity AgentЖ░ђ ВъљЖ▓Е ВдЮвфЁВЮё в░юЖИЅьЋўв»ђвАю,
EKS ьЂ┤вЪгВіцьё░ вѓ┤вХђ PodВЌљВёю Ж░ёьјИьЋўЖ▓ї AWS Вёюв╣ёВіцВЌљ ВаЉЖи╝ьЋ┤ВЋ╝ ьЋўвіћ Ж▓йВџ░ ВаЂьЋЕ

IAM Role for Service Account (IRSA) ВІювѓўвдгВўц

ВЃЂьЎЕ в╣ёЖ│а
EKS вѓ┤вХђВЌљВёю AWS Вёюв╣ёВіц ВаЉЖи╝ВЮ┤ ьЋёВџћьЋа вЋї Kubernetes ServiceAccountВЎђ IAM RoleВЮё 1:1 вДцьЋЉьЋўВЌг AWS вдгВєїВіц ВаЉЖи╝ВЮ┤ Ж░ђвіЦ
STS ВъёВІю ВъљЖ▓Е ВдЮвфЁВю╝вАю ВЮИВдЮьЋўЖ│а ВІХВЮё вЋї OIDC ьєаьЂ░Вю╝вАю STS AssumeRoleWithWebIdentityвЦ╝ ьўИВХюьЋўВЌг вІеЖИ░ ВъљЖ▓Е ВдЮвфЁ в░юЖИЅ
EKS вѓ┤ВЌљВёю вІцВѕўВЮў PodЖ░ђ AWS Вёюв╣ёВіцВЌљ ВаЉЖи╝ьЋ┤ВЋ╝ ьЋа вЋї Ж░Ђ PodВЌљ в│ёвЈёВЮў IAM RoleВЮё вХђВЌгьЋа ьЋёВџћ ВЌєВЮ┤ ServiceAccountвЦ╝ ВѓгВџЕьЋўВЌг IAM RoleВЮё Ж│хВюаьЋа Вѕў ВъѕВЮї
IAM ВаЋВ▒ЁВЮё вЇћ ВёИв░ђьЋўЖ▓ї Ж┤ђвдгьЋўЖ│а ВІХВЮё вЋї ьі╣ВаЋ Kubernetes Namespace, ServiceAccount вІеВюёвАю ВёИв░ђьЋю IAM ВаЋВ▒Ё ВаЂВџЕ Ж░ђвіЦ
EKSВЮў ЖИ░в│ИВаЂВЮИ AWS Вёюв╣ёВіц ВаЉЖи╝ в░ЕВІЮВЮ┤ ьЋёВџћьЋа вЋї IRSAвіћ EKSВЌљВёю AWS Вёюв╣ёВіцВЌљ ВаЉЖи╝ьЋа вЋї Ж░ђВъЦ ВЮ╝в░ўВаЂВю╝вАю ЖХїВъЦвљўвіћ в░ЕВІЮ

IRSA в░ЕВІЮВЮђ KubernetesВЮў ServiceAccountвЦ╝ AWS IAM RoleЖ│╝ ВДЂВаЉ ВЌ░Ж▓░ьЋўвіћ в░ЕВІЮВЮ┤в»ђвАю EKS вѓ┤вХђВЌљВёю AWS вдгВєїВіц(S3, DynamoDB, SQS вЊ▒)ВЌљ ВаЉЖи╝ьЋўвіћ PodвЦ╝ ьџеВюеВаЂВю╝вАю Ж┤ђвдгьЋа вЋї ВаЂьЋЕ

ВЃЂьЎЕв│ё Pod Identity vs IRSA

ВЃЂьЎЕ ВХћВ▓ю в░ЕВІЮ ВХћВ▓ю ВѓгВюа

EKS вѓ┤вХђВЌљВёю AWS Вёюв╣ёВіц ВаЉЖи╝ВЮ┤ ьЋёВџћьЋю Ж▓йВџ░ IRSA Kubernetes ServiceAccountвЦ╝ ьєхьЋ┤ IAM RoleВЮё ВъљвЈЎ ВЌ░Ж▓░ьЋа Вѕў ВъѕВќ┤ ьџеВюеВаЂ
STS ьўИВХю ВЌєВЮ┤ ВъљЖ▓Е ВдЮвфЁВЮё в░юЖИЅв░ЏЖ│а ВІХВЮђ Ж▓йВџ░ Pod Identity Pod Identity AgentЖ░ђ EKS Auth APIвАю ВъљЖ▓Е ВдЮвфЁВЮё в░юЖИЅьЋўВЌг STS ьўИВХюВЮ┤ вХѕьЋёВџћ
IAM RoleЖ│╝ Kubernetes вёцВъёВіцьјўВЮ┤Віцв│ё ВёИвХёьЎћвљю ЖХїьЋюВЮё ВаЂВџЕьЋўЖ│а ВІХВЮё вЋї IRSA Kubernetes ServiceAccountВЎђ IAM RoleВЮё 1:1 вДцьЋЉьЋўВЌг ВёИв░ђьЋю ЖХїьЋю Ж┤ђвдг Ж░ђвіЦ
  • EKS вѓ┤вХђВЌљВёю AWS Вёюв╣ёВіц ВаЉЖи╝ВЮё Ж┤ђвдгьЋўвіћ Ж░ђВъЦ ВЮ╝в░ўВаЂВЮИ в░ЕВІЮВЮђ IRSA
  • Pod Identityвіћ OIDC ВёцВаЋ ВЌєВЮ┤ Ж░ёьјИьЋўЖ▓ї ьЂ┤вЪгВіцьё░ вѓ┤вХђВЌљВёю ВЮИВдЮьЋа вЋї ьЎюВџЕ

В░ИЖ│а: ID ьјўвЇћваѕВЮ┤ВЁў / IDP в╣ёЖхљьЉю
