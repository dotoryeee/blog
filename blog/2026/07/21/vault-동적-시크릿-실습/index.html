Dynamic Secrets PostgreSQL Vault

Vault лҸҷм Ғ мӢңнҒ¬лҰҝ мӢӨмҠө

лӘ©н‘ң

  • Vaultмқҳ database secrets engineмңјлЎң DB кі„м •мқҙ мҡ”мІӯ мӢңм җм—җ л°ңкёүлҗҳкі  TTLмқҙ м§ҖлӮҳл©ҙ мһҗлҸҷмңјлЎң мӮ¬лқјм§ҖлҠ” кіјм •мқ„ лЎңм»¬м—җм„ң мӢӨмёЎн•ңлӢӨ
  • docker composeлЎң Vault(dev лӘЁл“ң)мҷҖ PostgreSQLмқ„ лқ„мҡ°кі , dotoryeee-app roleм—җм„ң TTL 2л¶„м§ңлҰ¬ кі„м •мқ„ л°ңкёүл°ӣм•„ к·ё кі„м •мңјлЎң мӢӨм ң psql м ‘мҶҚк№Ңм§Җ нҷ•мқён•ңлӢӨ
  • TTL л§ҢлЈҢмҷҖ lease revoke л‘җ кІҪлЎңлЎң кі„м •мқҙ нҡҢмҲҳлҗҳлҠ”м§Җ postgres мӘҪ role лӘ©лЎқмңјлЎң көҗм°Ё кІҖмҰқн•ңлӢӨ

лҸҷм Ғ мӢңнҒ¬лҰҝмқҳ к°ңл…җкіј Vault кө¬м„ұмҡ”мҶҢ(secret engine, auth method, policy, lease), мҝ лІ„л„ӨнӢ°мҠӨ м—°лҸҷ л°©мӢқмқҖ мӢңнҒ¬лҰҝ кҙҖлҰ¬ м •лҰ¬м—җм„ң лӢӨлӨҳлӢӨ. мқҙ кёҖмқҖ к·ё мң„м—җм„ң database м—”м§„мқ„ мӢӨм ңлЎң л¶ҷмқҙкі , мҡ”мІӯ н•ң лІҲм—җ кі„м •мқҙ мғқкІјлӢӨк°Җ мӢңк°„мқҙ м§ҖлӮҳл©ҙ м Җм ҲлЎң м—Ҷм–ҙм§ҖлҠ” кІғмқ„ лҲҲмңјлЎң нҷ•мқён•ҳлҠ” лҚ° м§‘мӨ‘н•ңлӢӨ.

мӢӨмҠө кө¬м„ұ

м»Ён…Ңмқҙл„Ҳ л‘җ к°ңлҘј н•ң compose л„ӨнҠёмӣҢнҒ¬м—җ мҳ¬лҰ°лӢӨ. VaultлҠ” database м—”м§„мңјлЎң postgresм—җ кҙҖлҰ¬мһҗ мһҗкІ©мңјлЎң л¶ҷм–ҙ мһҲлӢӨк°Җ, нҒҙлқјмқҙм–ёнҠёк°Җ мһҗкІ©мҰқлӘ…мқ„ мҡ”мІӯн•ҳл©ҙ к·ё мҲңк°„ мһ„мӢң roleмқ„ л§Ңл“Өм–ҙ лҸҢл ӨмӨҖлӢӨ. нқҗлҰ„мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

  • dotoryeee-vault: dev лӘЁл“ңлЎң лңЁлҠ” Vault м„ңлІ„. 8200м—җм„ң APIмҷҖ UIлҘј м—°лӢӨ
  • dotoryeee-postgres: лҸҷм Ғ кі„м •мқ„ л°ңкёүл°ӣмқ„ лҢҖмғҒ DB. dotoryeee_shop лҚ°мқҙн„°лІ мқҙмҠӨмҷҖ кҙҖлҰ¬мһҗ кі„м • dotoryeeeлҘј к°–лҠ”лӢӨ

мқҙлҜём§ҖлҠ” hashicorp/vault:1.18, postgres:16мңјлЎң кі м •н•ңлӢӨ. VaultлҠ” dev лӘЁл“ңлқј кё°лҸҷкіј лҸҷмӢңм—җ мһҗлҸҷмңјлЎң unsealлҗҳкі  лЈЁнҠё нҶ нҒ°мқҙ л°ңкёүлҗҳм§Җл§Ң, лҚ°мқҙн„°лҠ” л©”лӘЁлҰ¬м—җл§Ң лӮЁлҠ”лӢӨ.

Warning

рҹ’Ў dev лӘЁл“ң мў…лЈҢ мӢң м—”м§„ м„Өм •кіј мӢңнҒ¬лҰҝмқҙ лӘЁл‘җ мӮ¬лқјм§ҖлҜҖлЎң мӢӨмҠөВ·лҚ°лӘЁмҡ©мңјлЎңл§Ң м“ҙлӢӨ

лЁјм Җ compose нҢҢмқјмқ„ мһ‘м„ұн•ңлӢӨ. м»Ён…Ңмқҙл„Ҳ мқҙлҰ„кіј лҚ°лӘЁ нҒ¬лҰ¬лҚҙм…ңм—җ dotoryeeeлҘј л„Јм–ҙ UIмҷҖ л°ңкёү кі„м •м—җм„ң мһҗмЈј л…ём¶ңлҗҳкІҢ н•ңлӢӨ.

  dotoryeee-vault:                       #Vault м„ңлІ„ (dev лӘЁл“ң, лҚ°мқҙн„°лҠ” л©”лӘЁлҰ¬м—җл§Ң)
    image: hashicorp/vault:1.18

    environment:
      VAULT_DEV_ROOT_TOKEN_ID: dotoryeee-root      #dev лӘЁл“ң лЈЁнҠё нҶ нҒ°мқ„ кі м •к°’мңјлЎң м§Җм •

      VAULT_ADDR: http://127.0.0.1:8200            #м»Ён…Ңмқҙл„Ҳ м•Ҳ CLIк°Җ л°”лқјліј мЈјмҶҢ

      - "8200:8200"                       #UIмҷҖ APIлҘј нҳёмҠӨнҠё 8200мңјлЎң л…ём¶ң

  dotoryeee-postgres:                     #лҸҷм Ғ кі„м •мқ„ л°ңкёүл°ӣмқ„ лҢҖмғҒ DB

    container_name: dotoryeee-postgres
    environment:
      POSTGRES_DB: dotoryeee_shop         #мӢӨмҠөмҡ© лҚ°мқҙн„°лІ мқҙмҠӨлӘ…
      POSTGRES_USER: dotoryeee            #Vaultк°Җ кҙҖлҰ¬мһҗлЎң л¶ҷмқ„ кі„м • (superuser)

database secrets engine м„Өм •

database м—”м§„мқ„ нҷңм„ұнҷ”н•ңлӢӨ. нҷңм„ұнҷ”н•ҳл©ҙ database/ кІҪлЎңк°Җ мғқкё°кі , secrets listм—җм„ң нҷ•мқён•  мҲҳ мһҲлӢӨ.

мқҙм–ҙм„ң Vaultк°Җ postgresм—җ л¶ҷмқ„ м—°кІ°мқ„ м„Өм •н•ңлӢӨ. connection_urlмқҳ username, password мһҗлҰ¬лҠ” мқҙмӨ‘ мӨ‘кҙ„нҳё н…ңн”ҢлҰҝмңјлЎң л‘җкі , мӢӨм ң кҙҖлҰ¬мһҗ кі„м •кіј м•”нҳёлҠ” л’Өм—җ л”°лЎң л„ҳкёҙлӢӨ. allowed_rolesлЎң мқҙ м—°кІ°мқ„ м“ё мҲҳ мһҲлҠ” roleмқ„ dotoryeee-app н•ҳлӮҳлЎң м ңн•ңн•ңлӢӨ.

docker exec -e VAULT_TOKEN=dotoryeee-root dotoryeee-vault vault write database/config/dotoryeee-postgres \
  plugin_name=postgresql-database-plugin \
  allowed_roles="dotoryeee-app" \
  connection_url="postgresql://{{username}}:{{password}}@dotoryeee-postgres:5432/dotoryeee_shop?sslmode=disable" \
  username="dotoryeee" \
  password="dotoryeee-pass"
Success! Data written to: database/config/dotoryeee-postgres

connection_urlмқҳ hostлҘј dotoryeee-postgresлЎң м Ғмңјл©ҙ compose л„ӨнҠёмӣҢнҒ¬к°Җ м»Ён…Ңмқҙл„Ҳ мқҙлҰ„мқ„ DNSлЎң н’Җм–ҙмӨҖлӢӨ. VaultлҠ” м»Ён…Ңмқҙл„Ҳ м•Ҳм—җм„ң 5432лЎң л¶ҷмңјлҜҖлЎң нҳёмҠӨнҠё л§Өн•‘ нҸ¬нҠё 15432к°Җ м•„лӢҲлқј лӮҙл¶Җ нҸ¬нҠё 5432лҘј м“ҙлӢӨ.

лҸҷм Ғ кі„м •мқҙ мқҪмқ„ лҚ°лӘЁ н…Ңмқҙлё”мқ„ кҙҖлҰ¬мһҗ кі„м •мңјлЎң н•ҳлӮҳ л§Ңл“ лӢӨ. мқҙ н…Ңмқҙлё”мқҙ мһҲм–ҙм•ј л’Өм—җм„ң л°ңкёү кі„м •м—җ мӨҖ SELECT к¶Ңн•ңмқҙ нҷ•мқён•  лҢҖмғҒмқҙ мғқкёҙлӢӨ.

role л§Ңл“Өкё°

roleмқҖ "мһҗкІ©мҰқлӘ…мқ„ мҡ”мІӯл°ӣмңјл©ҙ postgresм—җм„ң л¬ҙмҠЁ SQLмқ„ мӢӨн–үн• м§Җ"лҘј м •мқҳн•ңлӢӨ. creation_statementsм—җ кі„м • мғқм„ұкіј к¶Ңн•ң л¶Җм—¬лҘј, revocation_statementsм—җ нҡҢмҲҳ SQLмқ„ м ҒлҠ”лӢӨ. default_ttlмқ„ 2л¶„мңјлЎң м§§кІҢ мӨҳм„ң л§ҢлЈҢлҘј кёҲл°© нҷ•мқён•  мҲҳ мһҲкІҢ н•ҳкі , max_ttlмқҖ 10л¶„мңјлЎң л‘”лӢӨ.

docker exec -e VAULT_TOKEN=dotoryeee-root dotoryeee-vault vault write database/roles/dotoryeee-app \
  db_name=dotoryeee-postgres \
  creation_statements="CREATE ROLE \"{{name}}\" WITH LOGIN PASSWORD '{{password}}' VALID UNTIL '{{expiration}}'; GRANT SELECT ON ALL TABLES IN SCHEMA public TO \"{{name}}\";" \
  revocation_statements="REVOKE ALL PRIVILEGES ON ALL TABLES IN SCHEMA public FROM \"{{name}}\"; DROP ROLE IF EXISTS \"{{name}}\";" \
  default_ttl="2m" \
  max_ttl="10m"
Success! Data written to: database/roles/dotoryeee-app

нҡҢмҲҳл¬ём—җ REVOKEлҘј лЁјм Җ л„ЈмқҖ лҚ°лҠ” мқҙмң к°Җ мһҲлӢӨ. мғқм„ұл¬ём—җм„ң кі„м •м—җ SELECTлҘј л¶Җм—¬н–Ҳкё° л•Ңл¬ём—җ, к¶Ңн•ңмқ„ к·ёлҢҖлЎң л‘” мұ„ DROP ROLEл§Ң н•ҳл©ҙ postgresк°Җ "dependent к°қмІҙк°Җ мһҲлӢӨ"л©° мӮӯм ңлҘј кұ°л¶Җн•ңлӢӨ. к·ёлһҳм„ң к¶Ңн•ңмқ„ лЁјм Җ нҡҢмҲҳн•ҳкі  roleмқ„ м§ҖмҡҙлӢӨ.

Warning

рҹ’Ў к¶Ңн•ңмқ„ л¶Җм—¬н•ң roleмқҖ DROP ROLEл§ҢмңјлЎң мӮӯм ңлҗҳм§Җ м•ҠмңјлҜҖлЎң к¶Ңн•ң нҡҢмҲҳлҘј м•һм—җ л‘”лӢӨ

role м„Өм •мқҙ л“Өм–ҙк°„ кІғмқ„ readлЎң нҷ•мқён•ңлӢӨ.

м—¬кё°к№Ңм§Җмқҳ м„Өм •мқҖ Vault UIм—җм„ңлҸ„ к·ёлҢҖлЎң ліҙмқёлӢӨ. 8200м—җ лЈЁнҠё нҶ нҒ°мңјлЎң лЎңк·ёмқён•ңлӢӨ. MethodлҠ” Token, Token м№ём—җ dotoryeee-rootлҘј л„ЈлҠ”лӢӨ.

лЎңк·ёмқён•ҳл©ҙ л°©кёҲ нҷңм„ұнҷ”н•ң database м—”м§„кіј, л’Өм—җм„ң м“ё dotoryeee-kv м—”м§„мқҙ лӘ©лЎқм—җ ліҙмқёлӢӨ.

database м—”м§„мқҳ dotoryeee-app role мғҒм„ём—җлҠ” TTLкіј мөңлҢҖ TTL, к·ёлҰ¬кі  мғқм„ұл¬ёВ·нҡҢмҲҳл¬ёмқҙ к·ёлҢҖлЎң л…ём¶ңлҗңлӢӨ.

лҸҷм Ғ кі„м • л°ңкёүкіј м ‘мҶҚ
